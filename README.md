# Стек

- Nest.js (Zod-валидация)
- Next.js (Ant Design UI библиотека)
- MySQL (PrismaORM)
- Redis
- Docker Compose
- Caddy (сервер)
- Swagger (документация)
- Husky

** Почему выбран этот стек?**

Nest.js предлагает из коробки прекрасную архитектуру и полностью типизирована. Быстрый запуск и хорошее поддерживание проекта(При соблюдении архитектуры).

Next.js - Указан как обязательный инструмент из ТЗ. Я бы выбрал Nuxt, по той же причине. Более регламентированная структура, friendly синтаксис(особенно для верстки) и автоимпорт

MySQL - Стандартный выбор.

Redis - Здесь храним сессии авторизованных пользователей.

Docker Compose - Быстрый CI/CD. Дописал Make для удобства.

Caddy - Можно было использовать nginx-certbot контейнер, но там нужен домен для lets-encrypt(возможно есть тестовые серты, но не работал).

Swagger - автодокументация. Доступна только если тестовое окружение.

Husky - хук на гит коммит. Автозапуск eslint + prettier.

## Запуск

```bash
#запуск
$ make up

#запускаем сид
$ make seed
```

Сайт - https://localhost/ браузер будет чуть ругаться "Перейти на сайт localhost (небезопасно)"

Документация - http://localhost:3000/api/docs

Сид создает 3 учетные записи и "рандомно" заполняет журнал. Только владелец своей записи может ее редактировать.

Логин - `user1@example.com` | `user2@example.com` | `user3@example.com`

Пароль - `password`

## Примечание

Гит вынесен наверх для удобства, что бы не разносить по 3 репозиториям. В продакшн мы бы вынесли все в отдельные репо и разделили бы докер контейнеры на группы, что бы при автодеплое не пересобирать все.
